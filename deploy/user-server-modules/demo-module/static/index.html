功能概述

自定义模块功能允许用户扩展 deepseek-cowork 服务端能力，无需修改核心代码。主要特点：

  • 独立的模块目录，便于管理和版本控制
  • 标准化的模块接口，简化开发流程
  • 支持路由注册、静态资源、事件发射
  • 热加载支持，服务重启即可生效

API 接口

  • GET /demo/ 本介绍页面
  • GET /api/demo/status 获取模块运行状态
  • POST /api/demo/echo Echo 请求体
  • GET /api/demo/info 获取模块详细信息

模块接口规范

自定义模块需要导出一个 setup 函数，返回实现以下接口的服务实例：

方法 必须 说明
init() 可选 初始化逻辑，在模块加载后调用
setupRoutes(app) 可选 注册 Express 路由
start() 可选 启动服务
stop() 可选 停止服务，清理资源

快速开发指南

创建模块目录
# Windows %APPDATA%\deepseek-cowork\user-server-modules\my-module\ # macOS ~/Library/Application Support/deepseek-cowork/user-server-modules/my-module/ # Linux ~/.config/deepseek-cowork/user-server-modules/my-module/
编写模块入口 (index.js)
const { EventEmitter } = require('events'); function setupMyModuleService(options = {}) { class MyModuleService extends EventEmitter { async init() { console.log('[MyModule] 初始化'); } setupRoutes(app) { app.get('/api/my-module/hello', (req, res) => { res.json({ message: 'Hello from my module!' }); }); } async start() { this.emit('started'); } async stop() { this.emit('stopped'); } } return new MyModuleService(); } module.exports = { setupMyModuleService };
创建配置文件 (userServerModulesConfig.js)
module.exports = { modules: [ { name: 'my-module', module: './my-module', setupFunction: 'setupMyModuleService', enabled: true, features: { hasRoutes: true } } ] };
重启服务

服务重启后会自动加载新模块，访问 /api/my-module/hello 验证。

部署命令

# 查看可用模块模板 node deploy/index.js module --list # 部署演示模块 node deploy/index.js module demo-module # 查看已部署模块状态 node deploy/index.js module --status # 移除已部署模块 node deploy/index.js module --remove demo-module
